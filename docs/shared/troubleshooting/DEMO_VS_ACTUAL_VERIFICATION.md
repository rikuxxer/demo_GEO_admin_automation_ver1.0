# デモ画面と実画面の実装比較（検証結果）

操作ガイドで使用するデモ画面（`DemoScreen.tsx`）が、実際のアプリ画面とどの程度一致しているかを検証した結果です。

## 検証日・対象

- **検証対象**: `src/components/DemoScreen.tsx` の各 type（`projects`, `project-form`, `project-detail`, `bulk-import`）
- **比較先**: 実画面の `ProjectForm`, `ProjectDetail`, `SummaryCards`, `ProjectTable`, `BulkImport` および `App.tsx` の案件サマリレイアウト

---

## 1. 案件一覧・案件登録フォーム（`projects` / `project-form`）

| 項目 | 実物 | デモ（修正後） | 一致 |
|------|------|----------------|------|
| ページヘッダー | 「案件サマリ」+ 縦線 | 同様 | ✓ |
| 新規依頼ボタン | 「案件の新規依頼」+ ドロップダウン（手動で登録 / 一括登録） | 同様 | ✓ |
| SummaryCards | 7枚・担当案件数/下書き/入力不備あり/連携依頼待ち/連携依頼済/連携完了/期限切れ間近 | 同様の7枚・文言・スタイル | ✓ |
| ProjectTable | 案件一覧ヘッダー、検索、列（案件ID/登録日/広告主/訴求/担当者/Status）、フッター | 同様（フッターは「Items per page」+ 件数 + ページネーション風） | ✓ |
| 案件登録モーダル | タイトル「新規案件登録」、サブタイトル、基本情報/UNIVERSE/配信/その他、キャンセル・登録する | 同様（ボタン文言「登録する」、備考 placeholder 実物に合わせて修正済み） | ✓ |

**修正済み（実物に合わせた箇所）**

- **SummaryCards デモ**: 選択状態のスタイル（ring-2 ring-[#5b5fff]、選択時チェックマーク・文字色）を実装。各カードにツールチップ（Info アイコン＋ホバーで説明文）を追加。`selectedStatus` を props で渡すとそのカードが選択表示される。
- **ProjectTable デモ**: ヘッダーに Checkbox 列・各列をソート風ボタン（ChevronDown）に変更。検索横に「自分の案件のみ」ボタンを追加。各行の先頭に Checkbox を追加。

## 5. 今回の修正内容（実物に合わせた箇所）

1. **ProjectForm デモ**
   - 送信ボタン: 「登録」→「**登録する**」
   - 備考の placeholder: 「特記事項、注意点など」→「**特記事項、注意点、連絡事項などを記載してください**」

2. **BulkImport デモ（type='bulk-import'）**
   - 説明カードに「**⚠️** 複数案件を…」の表記を追加。
   - 「**※ プルダウンで簡単入力。広告主や代理店の方も入力しやすい形式です**」の1行を追加。

3. **デモ内の一括登録モーダル（projects 画面上で開く場合）**
   - 説明カードを実物と同じリスト形式＋⚠️＋※の文言に統一。

4. **案件テーブル（デモ）**
   - フッターを実物に合わせて「Items per page」の select ＋ 件数表示 ＋ ページネーション風（1 of 1 pages + 前後ボタン）に変更。

5. **再確認時の追加修正**
   - **案件詳細デモ（overview タブ）**: 実物と同様に「備考」セクションを追加（配信情報の次、アイコン＋「備考」見出し＋「備考・メモ」ラベル＋値「-」）。
   - **一括登録デモ（type='bulk-import' およびモーダル内）**: 説明カードの②案件情報で「1案件のみ登録可能」を実物と同じ `<span className="text-red-600 font-bold">` で赤太字に統一。

6. **「確認した内容をすべて修正」で実施した内容**
   - **SummaryCards デモ**: 実物と同様に選択状態の見た目（ring-2 ring-[#5b5fff]、選択時チェックマーク・文字色）を実装。各カードにツールチップ（Info アイコン＋ホバーで説明文）を追加。`selectedStatus` props で選択カードを指定可能。
   - **ProjectTable デモ**: 実物と同様に Checkbox 列（ヘッダー・各行）を追加。各列ヘッダーをソート風ボタン（ChevronDown）に変更。検索横に「自分の案件のみ」ボタンを追加。
   - **案件詳細デモ**: 実物と同様に「連絡事項」タブ（messages）を追加し、4タブ構成に。連絡事項タブのコンテンツはプレースホルダー表示。

---

## 6. まとめ

- **文言・レイアウト**: 案件一覧・案件登録フォーム・案件詳細（3タブ）・一括登録はいずれも実物に合わせてあり、今回の修正でさらに一致させた。
- **機能面**: デモは「見た目」と「操作ガイドで指し示す要素」に特化しており、フィルタ・ソート・メッセージ/変更履歴タブなどは意図的に省略している。
- **「全く同じ」の範囲**: 見た目・文言・タブ数（案件詳細は4タブ）・SummaryCards の選択状態・ツールチップ・ProjectTable の Checkbox・ソート風ヘッダー・「自分の案件のみ」まで実物に合わせてある。変更履歴タブ（admin 専用）のみデモでは省略。
